## 1. نظرة عامة | Executive Summary

### 1.1 الهدف من النظام | System Objective

بناء نظام متكامل لإدارة الرواتب والموارد البشرية مصمم خصيصاً للسوق السعودي والخليجي، يتميز بـ:

- محرك حساب رواتب ذكي متوافق مع قوانين العمل السعودية
- تكامل كامل مع منصة Zoho
- ذكاء اصطناعي مدمج للتحليل والتنبؤ
- بوابة موظف للخدمة الذاتية
- أمان على مستوى البنوك المركزية

### 1.2 نطاق المشروع | Project Scope

#### داخل النطاق (In Scope):
- إدارة بيانات الموظفين الكاملة
- معالجة مسيرات الرواتب الشهرية
- حساب الاستحقاقات والاستقطاعات
- تكامل مع Zoho People و Zoho Payroll
- بوابة موظف للاطلاع على القسائم
- نظام حضور وانصراف
- إدارة السلف والقروض
- تقارير تحليلية متقدمة
- مساعد ذكاء اصطناعي
- تطبيق جوال (iOS & Android)

#### خارج النطاق (Out of Scope):
- إدارة المشاريع الداخلية
- نظام CRM منفصل
- إدارة المخزون
- نقاط البيع (POS)

## 2. المتطلبات الوظيفية | Functional Requirements

### 2.1 إدارة الموظفين | Employee Management

#### FR-001: تسجيل موظف جديد
**الوصف:** القدرة على إضافة موظف جديد بكامل بياناته  
**الأولوية:** عالية (High)  
**المدخلات:**
- رقم الموظف (Employee ID)
- الاسم الكامل (عربي + إنجليزي)
- الجنسية
- رقم الهوية/الإقامة
- القسم
- المسمى الوظيفي
- تاريخ التوظيف
- نوع العقد (دائم/مؤقت/متعاقد)
- نوع الدوام (كامل/جزئي)
- الراتب الأساسي
- بيانات الحساب البنكي (مشفرة)

**المخرجات:**
- رقم موظف فريد
- رسالة تأكيد الإضافة
- إنشاء حساب بوابة الموظف

**قواعد العمل:**
- رقم الموظف يجب أن يكون فريداً
- رقم الهوية/الإقامة لا يمكن تكراره
- الراتب الأساسي يجب أن يكون أكبر من صفر
- تاريخ التوظيف لا يمكن أن يكون في المستقبل

#### FR-002: تعديل بيانات موظف
**الوصف:** تحديث بيانات موظف موجود  
**الأولوية:** عالية (High)  
**القيود:**
- لا يمكن تعديل رقم الموظف
- تسجيل كل تعديل في سجل المراجعة (Audit Log)
- تتبع تاريخ التعديلات

#### FR-003: إنهاء خدمة موظف
**الوصف:** تغيير حالة الموظف إلى "منتهي الخدمة"  
**المتطلبات:**
- حساب نهاية الخدمة والمستحقات
- إيقاف الراتب الشهري
- إغلاق الوصول لبوابة الموظف
- الاحتفاظ بالبيانات التاريخية

### 2.2 هيكل الرواتب | Salary Structure

#### FR-010: تكوين مكونات الراتب
**الوصف:** إنشاء وإدارة مكونات الراتب المرنة  
**الأولوية:** عالية (High)  

**أنواع المكونات:**

**الاستحقاقات (Earnings):**
- الراتب الأساسي (Basic Salary)
- بدل السكن (Housing Allowance)
- بدل النقل (Transportation Allowance)
- بدل الاتصالات (Communication Allowance)
- العمل الإضافي (Overtime)
- المكافآت (Bonus)
- العمولات (Commission)
- الحوافز (Incentives)
- بدلات أخرى (Other Allowances)

**الاستقطاعات (Deductions):**
- التأمينات الاجتماعية - GOSI (Social Insurance)
- خصومات الغياب (Absence Deductions)
- خصومات التأخير (Late Deductions)
- السلف (Advances)
- القروض (Loans)
- الغرامات (Penalties)
- خصومات أخرى (Other Deductions)

**أنواع الحساب:**
- مبلغ ثابت (Fixed Amount)
- نسبة مئوية من الأساسي (Percentage of Basic)
- معادلة حسابية (Formula-based)

#### FR-011: ربط هيكل راتب بموظف
**الوصف:** تخصيص مكونات راتب محددة لكل موظف  
**المتطلبات:**
- دعم تواريخ السريان (Effective Dates)
- السماح بتغيير الهيكل مستقبلاً
- الاحتفاظ بالتاريخ الكامل للتغييرات

### 2.3 معالجة الرواتب | Payroll Processing

#### FR-020: إنشاء مسير رواتب شهري
**الوصف:** معالجة رواتب جميع الموظفين النشطين لفترة محددة  
**الأولوية:** حرجة (Critical)  

**خطوات المعالجة:**
1. تحديد فترة الدفع (Pay Period)
2. جلب الموظفين النشطين
3. حساب الاستحقاقات لكل موظف
4. حساب الاستقطاعات
5. حساب الصافي (Net Salary)
6. إنشاء قسائم الرواتب (Payslips)
7. المراجعة والاعتماد
8. الدفع للحسابات البنكية

**قواعد الحساب:**

**التأمينات الاجتماعية (GOSI):**
- موظفين سعوديين: 10% من الراتب الأساسي
- موظفين غير سعوديين: 2% من الراتب الأساسي

**العمل الإضافي:**
- سعر الساعة = الراتب الأساسي ÷ (30 يوم × 8 ساعات)
- أجر العمل الإضافي = سعر الساعة × 1.5
- أجر العمل في أيام الراحة = سعر الساعة × 2

**خصم الغياب:**
- سعر اليوم = الراتب الأساسي ÷ 30
- خصم الغياب = عدد أيام الغياب × سعر اليوم

**خصم التأخير:**
- 3 مرات تأخير = خصم ساعة واحدة
- سعر الساعة = الراتب الأساسي ÷ 240

#### FR-021: اعتماد مسير الرواتب
**الوصف:** مراجعة واعتماد الرواتب قبل الدفع  
**الصلاحيات:**
- مدير الموارد البشرية
- المدير المالي
- المدير العام

**المتطلبات:**
- عرض ملخص إجمالي
- عرض أي شذوذات أو أخطاء محتملة
- إمكانية تعديل قبل الاعتماد النهائي
- تسجيل من اعتمد ومتى

#### FR-022: دفع الرواتب
**الوصف:** تحويل الرواتب للحسابات البنكية  
**الطرق المدعومة:**
- ملف WPS (Wage Protection System)
- تكامل API مباشر مع البنوك
- دفعات جماعية (Bulk Payments)
- STC Pay للدفع الفوري

### 2.4 بوابة الموظف | Employee Portal

#### FR-030: لوحة تحكم الموظف
**الوصف:** واجهة شخصية لكل موظف  
**الأولوية:** عالية (High)  

**العناصر المعروضة:**
- ملخص الراتب الحالي
- آخر قسيمة راتب
- رصيد الإجازات
- سجل الحضور والانصراف
- السلف والقروض النشطة
- الإشعارات المهمة

#### FR-031: عرض قسائم الرواتب
**الوصف:** الاطلاع على قسائم الرواتب السابقة  
**الميزات:**
- قائمة بجميع القسائم (آخر 12 شهر)
- عرض تفصيلي لكل قسيمة
- تحميل PDF
- إرسال عبر البريد الإلكتروني
- مشاركة عبر WhatsApp

**محتوى القسيمة:**
- معلومات الشركة والموظف
- فترة الدفع
- جدول الاستحقاقات بالتفصيل
- جدول الاستقطاعات بالتفصيل
- الإجمالي والصافي
- تاريخ الدفع
- توقيع إلكتروني

#### FR-032: طلب إجازة
**الوصف:** تقديم طلب إجازة إلكترونياً  
**الأنواع المدعومة:**
- إجازة سنوية
- إجازة مرضية
- إجازة طارئة
- إجازة بدون راتب
- إجازة أمومة/أبوة

**سير العمل:**
1. الموظف يختار النوع والفترة
2. النظام يحسب الرصيد المتاح
3. إرسال للمدير المباشر
4. إشعار بالموافقة/الرفض
5. تحديث رصيد الإجازات

#### FR-033: طلب سلفة
**الوصف:** طلب سلفة على الراتب  
**القيود:**
- الحد الأقصى: 50% من الراتب الأساسي
- التسديد: على أقساط شهرية
- لا يمكن طلب سلفة جديدة قبل سداد السابقة

## 3. المتطلبات غير الوظيفية | Non-Functional Requirements

### 3.1 الأداء | Performance

#### NFR-001: زمن الاستجابة
- صفحات الويب: < 2 ثانية
- API calls: < 100ms
- معالجة مسير رواتب (100 موظف): < 30 ثانية
- تحميل تقرير PDF: < 3 ثوانٍ

#### NFR-002: السعة
- دعم حتى 10,000 موظف
- 1,000 مستخدم متزامن
- تخزين 5 سنوات من البيانات
- معالجة 50 مسير رواتب متزامن

#### NFR-003: التوافر
- Uptime: 99.9% (8.76 ساعة توقف سنوياً)
- نافذة الصيانة: يوم الجمعة 2-4 صباحاً
- Backup تلقائي يومياً
- Disaster Recovery Plan جاهز

### 3.2 الأمان | Security

#### NFR-010: المصادقة والتفويض
- Multi-Factor Authentication (MFA) إلزامي
- Role-Based Access Control (RBAC)
- Session timeout: 15 دقيقة خمول
- Password policy: 12 حرف على الأقل
- تسجيل محاولات الدخول الفاشلة

#### NFR-011: التشفير
- HTTPS/TLS 1.3 لكل الاتصالات
- AES-256 للبيانات الحساسة
- بيانات البنوك مشفرة في قاعدة البيانات
- كلمات المرور: bcrypt + salt

#### NFR-012: سجل المراجعة
- تسجيل كل عملية (Create/Update/Delete)
- تسجيل: المستخدم، الوقت، IP، التغييرات
- سجلات غير قابلة للحذف (Immutable)
- الاحتفاظ بالسجلات 7 سنوات

#### NFR-013: الامتثال
- GDPR compliance
- Saudi Labor Law compliance
- ISO 27001 certified
- اختبار اختراق ربع سنوي

### 3.3 قابلية الاستخدام | Usability

#### NFR-020: الواجهة
- دعم العربية والإنجليزية
- تبديل اللغة فوري
- تصميم متجاوب (Responsive)
- دعم Dark Mode
- إمكانية الوصول (WCAG 2.1 Level AA)

#### NFR-021: التوافق
**المتصفحات:**
- Chrome 100+
- Firefox 100+
- Safari 15+
- Edge 100+

**الأجهزة:**
- Desktop (Windows, macOS, Linux)
- Tablet (iPad, Android)
- Mobile (iOS 15+, Android 11+)

### 3.4 قابلية التوسع | Scalability

#### NFR-030: Horizontal Scaling
- إضافة خوادم جديدة بسهولة
- Load balancing تلقائي
- Database sharding عند الحاجة
- Microservices architecture جاهزة

#### NFR-031: Vertical Scaling
- دعم زيادة موارد الخادم
- تحسين الاستعلامات تلقائياً
- Caching متعدد المستويات

### 3.5 قابلية الصيانة | Maintainability

#### NFR-040: الكود
- Clean Code principles
- SOLID principles
- Unit Tests: 80%+ coverage
- Integration Tests للـ APIs الحرجة
- Documentation كاملة

#### NFR-041: المراقبة
- Application Performance Monitoring (APM)
- Error tracking (Sentry)
- Log aggregation (ELK Stack)
- Alerts تلقائية للمشاكل

## 15. الملحقات | Appendices

### ملحق A: مسرد المصطلحات

**WPS:** Wage Protection System - نظام حماية الأجور السعودي  
**GOSI:** General Organization for Social Insurance - المؤسسة العامة للتأمينات الاجتماعية  
**MFA:** Multi-Factor Authentication - المصادقة متعددة العوامل  
**API:** Application Programming Interface  
**JWT:** JSON Web Token  
**RBAC:** Role-Based Access Control  
**SLA:** Service Level Agreement

### ملحق B: معادلات الحساب

**الراتب الإجمالي (Gross Salary):**
```
Gross = Basic Salary + Sum(All Earnings)
```

**الراتب الصافي (Net Salary):**
```
Net = Gross - Sum(All Deductions)
```

**التأمينات الاجتماعية:**
```
GOSI = Basic Salary × Rate
where Rate = 0.10 (Saudi) or 0.02 (Non-Saudi)
```

**العمل الإضافي:**
```
Overtime Pay = (Basic Salary ÷ 240) × Hours × 1.5
```

**خصم الغياب:**
```
Absence Deduction = (Basic Salary ÷ 30) × Days
```
